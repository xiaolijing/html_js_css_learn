作为前端，什么是伪类，什么是伪元素？

概念上的区别

伪类，更多的定义元素所处的状态。常见伪类有：:hover，:active，:focus，:visited，:link，:not，:first-child，:last-child
伪元素，指不存在于dom树中的虚拟元素，可以像正常的html元素一样定义css，却无法用js获取对象。常见的伪元素有： ::before，::after，::first-letter，::first-line

Css3明确规定了，伪类用一个冒号（:）来表示，而伪元素则用两个冒号(::)来表示。

这其中有兼容性问题，它们的写法可以是一致的，都用一个冒号表示。所以非常容易混淆

实战场景——伪类

表单校验

表单的校验中，常会用到 :required、:valid 和 :invalid 这三个伪类。先来看看它们所代表的含义。

  • :required，指定具有 required属性 的表单元素
  • :valid，指定一个 匹配指定要求 的表单元素
  • :invalid，指定一个 不匹配指定要求 的表单元素
  • :checked，有checked属性的控件

元素的index

执行一些列标签中的某个元素，可以用:nth-child(n)与:nth-of-type(n)来找到并指定样式。

  • :first-of-type 选择每个p元素是其父级的第一个p元素
  • :last-of-type 选择每个p元素是其父级的最后一个p元素
  • :only-of-type 选择每个p元素是其父级的唯一p元素
  • :only-child 选择每个p元素是其父级的唯一子元素
  • :nth-child(n) 选择每个p元素是其父级的第二个子元素
  • :nth-last-child(n 选择每个p元素的是其父级的第二个子元素，从最后一个子项计数
  • :nth-of-type(n) 选择每个p元素是其父级的第二个p元素
  • :nth-last-of-type(n) 选择每个p元素的是其父级的第二个p元素，从最后一个子项计数
  • :last-child 选择每个p元素是其父级的最后一个子级。

注意：n可以是大于零的数字，或者类似2n+1的表达式，或者even（奇）/odd（偶）

区别：

  • :XXX-type(n) 除了关注n之外，还需要关注最前面的类型，也就是标签。即同一个父容器中，按照定义的标签元素来计数
  • :XXX-child(n) 它关注的是：其父元素下的第n个孩子，与类型无关。
